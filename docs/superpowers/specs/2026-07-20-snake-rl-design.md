# Snake-RL — обучаемая змея в непрерывном 2D-мире

**Дата:** 2026-07-20
**Статус:** дизайн утверждён + пройден код-ревью (C1–C3, I1–I8, M1–M9 учтены), готов к плану реализации

## Идея

2D-мир как полноценная игра: змея с непрерывным углом движения (не только
4 направления), мир замыкается тором (лево↔право, верх↔низ). Змеёй управляет
ИИ, обученный с подкреплением (RL, PPO). Два режима: **train** (headless,
наблюдаем, как M1 оптимально обучает змею) и **watch** (pygame, смотрим на
обученную змею в свежем рандомном мире). Миры всегда случайные — змея учится
выживать в любом.

Змея не знает карту — только органы чувств: зрение (рей-кастинг) и обоняние
(запах от куриц, слабеет с расстоянием, блокируется препятствиями). Ест куриц
(растёт), умирает от столкновения с камнем/деревом или с собой. Есть рывок,
ограниченный выносливостью.

## Решения (согласованы с пользователем)

- **Стек:** Python 3.13 + PyTorch + PPO (Stable-Baselines3) + Gymnasium + pygame.
- **Управление:** дискретное — `MultiDiscrete([3, 2])`: руление `{−Δ°, прямо, +Δ°}` × рывок `{нет, да}`.
- **Рывок и выносливость:** рывок разгоняет змею (тратит выносливость), выносливость восстанавливается в покое. Балансом гарантирован догон убегающей курицы (инвариант ниже).
- **Голод:** не убивает (строго по ТЗ), но **не декоративен** — чем голоднее, тем выше пошаговый штраф (даёт срочность). Смерть — только от столкновений.
- **Обоняние:** упрощённое — сила ~ `1/(1+r)`, блокируется препятствиями (прямая видимость).
- **Наблюдение:** train headless на полной скорости + текстовая статистика; живая картинка — отдельный watch на актуальном чекпоинте.

## Ключевые константы (единый конфиг `config.py`)

Все величины в единицах/шаг, шаг симуляции `dt = 1`. Это **единственный** источник
чисел; при старте `env` проверяет инварианты (assert) — миска́либрованный билд
падает сразу, а не «молча плохо учится».

| Константа | Значение (старт) | Роль |
|---|---|---|
| `world_size` | случайно `[60, 100]` по каждой оси | размер тор-мира на эпизод |
| `v_snake` | 1.0 | обычная скорость головы |
| `v_dash` | 2.0 | скорость на рывке |
| `v_wander` | 0.25 | скорость блуждания курицы |
| `v_flee` | 1.15 | скорость убегания курицы (`> v_snake` — пешком не догнать) |
| `R_flee` | 12 | радиус вспугивания курицы |
| `turn_deg` (Δ°) | 8° | поворот за шаг |
| `S_max` / `drain` / `regen` | 30 / 1.0 / 0.3 | выносливость: запас / расход на рывке / восстановление в покое |
| `head_radius` | 0.8 | радиус головы (столкновения) |
| `body_radius` | 0.5 | полутолщина тела |
| `chicken_radius` | 0.6 | радиус курицы |
| `eat_radius` | `head_radius + chicken_radius` = 1.4 | дистанция поедания |
| `segment_spacing` | 0.6 | шаг между точками тела вдоль следа |
| `obstacle_radius` | случайно `[1.5, 4.0]` | камни/деревья |
| `energy_max` / `energy_decay` / `energy_refill` | 100 / 0.05 / 40 | голод: запас / расход-за-шаг / доливка за курицу |
| `episode_horizon` T | 2000 | предел шагов эпизода (truncation) |
| `n_rays` / `fov` | 9 / ±135° | зрение (270° охват, слепа только узкая зона строго сзади) |
| `frame_stack` | 4 | склейка кадров (память, см. POMDP) |
| `ray_range` | 20 | дальность лучей и «дистанция нормировки» |

### Инвариант достижимости задачи (assert при `env.__init__`)

Держится **по построению** и проверяется тестом:
```
(1) v_dash > v_flee                                  # рывком в принципе быстрее убегающей курицы
(2) (S_max / drain) · (v_dash − v_flee) ≥ k · R_flee # запаса выносливости хватает с запасом (k≥1.5)
(3) turn_deg / 2 < atan(eat_radius / R_flee)         # прицельная точность: можно навестись в «коридор поедания»
```
С текущими числами: (1) 2.0 > 1.15 ✓; (2) `30·0.85 = 25.5 ≥ 1.5·12 = 18` ✓; (3) `4° < atan(1.4/12) = 6.65°` ✓.
Запас `k` (не «впритык») закрывает потери на разворот к цели в 2D — идеально
колинеарный рывок не требуется. Смысл: подкрасться на обычной скорости, выровняться
на курицу, дать прямой рывок с ~`R_flee`.

## Мир и физика

- **Континуальный тор:** координаты float, `x % W`, `y % H`. Размер рандомный на эпизод (домен-рандомизация → змея не переобучается под один размер; при этом наблюдение эгоцентрично и фикс-длины, так что `VecEnv` из разноразмерных миров легален).
- **Геометрия «ближайшего образа» (везде!):** всякий парный вектор между объектами
  берётся через ближайший образ через шов тора:
  `d_axis = ((a − b + L/2) mod L) − L/2`. Это касается **всего**: дистанций,
  рей-кастинга (луч продолжается за швом — марш по модулю или проверка против
  3×3 тайл-образов объектов в пределах `ray_range`), прямой видимости для запаха,
  поиска ближайшей курицы, направления убегания и градиента запаха. *Без этого шов
  тора — сквозной баг восприятия, действий и reward одновременно.*
- **Змея:** голова движется под углом `heading` со скоростью `v_snake`/`v_dash`.
  Тело — след из точек через `segment_spacing`. Рост от еды = увеличение целевой
  длины. Тело держим ограниченным (перф рей-кастинга и сложность POMDP растут с длиной).
- **Столкновения — заметающая проверка (swept):** на рывке шаг ≈2× длиннее, поэтому
  проверяем не точку-голову, а **отрезок `[prev_head → new_head]`** против кругов
  (препятствия, сегменты тела) — иначе голова «протуннелит» сквозь тонкое
  препятствие/своё тело между тиками и смерть не засчитается (а политика этим
  начнёт читерить). Альтернатива — саб-степпинг рывка на `k=2–4` под-шага.
- **Столкновение с собой:** отрезок головы против сегментов тела, **пропуская**
  сегменты в пределах дуговой длины `head_radius + segment_spacing + body_radius`
  от головы (они всегда рядом; порог по дуге, а не «N ближайших» — число сегментов
  в одном шаге меняется с рывком).
- **Препятствия (камни/деревья):** статичные круги. Поведение идентично (оба убивают
  касанием головы) — одна сущность с флагом типа только для отрисовки. *ponytail: не плодить два класса ради спрайтов.*
- **Куры:** блуждание (`v_wander`, редкая смена направления). Если змея ближе `R_flee` —
  бегут прочь со скоростью `v_flee > v_snake` (пешком не догнать). Съедена = голова в
  `eat_radius` → змея растёт, энергия +`energy_refill`.
  - **Спавн:** поддерживаем ≥1 курицы всегда; новые появляются в среднем раз в ~200
    шагов, пока `< 5`. Позиция — rejection sampling против препятствий/тела/головы
    (курица не рождается «в камне» или под змеёй).
- **Смерть:** только касание головой препятствия или собственного тела. О «стену» не умирает (мир-тор).
- **Конец эпизода:** смерть (`terminated`) **или** достижение `episode_horizon` T
  (`truncated`). При truncation значение бутстрапится (GAE), при смерти — нет; SB3
  `TimeLimit` + корректная обработка `truncated` (иначе выжившая змея никогда не
  сбрасывается и глушит разнообразие миров в rollout).

### Рывок и выносливость

- `stamina ∈ [0, S_max]`, старт полный.
- **Рывок активен** (`dash=1` и `stamina > 0`): скорость `v_dash`, трата `drain`/шаг.
- **В покое** (`dash=0` или пусто): скорость `v_snake`, восстановление `regen`/шаг (`regen < drain`).
- Угловая скорость `turn_deg`/шаг одна и та же → на рывке радиус поворота больше (реалистично). `stamina` есть в наблюдении → «рывок при пустой выносливости игнорируется» остаётся марковским, а не скрытым нестационарным эффектом.

## Органы чувств (вектор наблюдения)

Всё эгоцентрично (в системе координат головы). Змея карту не знает. Базовый вектор —
**42 float**, затем `VecFrameStack(4)` даёт сети временну́ю память (168 входов).

- **Зрение — 9 лучей** веером ±135° (270° охват; слепа только узкая зона строго сзади).
  Каждый луч: нормированная дистанция `hit/ray_range ∈ [0,1]` + тип one-hot
  `{препятствие, курица, своё тело}`. Пустой луч (нет попадания в `ray_range`):
  дистанция = 1.0, тип = все нули. → 9 × 4 = **36**.
- **Обоняние → 3:** интенсивность в точке головы + локальный градиент (2 компоненты
  в системе головы).
  - Интенсивность = сумма по **видимым** курицам `1/(1+r)` (ограничена, не взрывается
    при `r→0`; курица за препятствием даёт 0 — «за камнем не пахнет»).
  - Градиент = аналитическая сумма `∇[1/(1+r)]` по видимым курицам (не конечная
    разность — избегаем разрывного шума на границе видимости), повёрнута в систему головы.
- **Проприоцепция → 3:** энергия `E/energy_max`, длина тела `len/len_cap` (нормирована
  на потолок), выносливость `stamina/S_max`.

Слепые зоны (узкая корма) закрывает временна́я память (frame-stack) — сеть помнит,
куда ушло тело, что критично при смерти от самопересечения. Итого 42 → 168.

**Нормировка/стабильность:** все входы ограничены по построению (дистанции ∈ [0,1],
запах `1/(1+r) ∈ (0,1]`, длина/энергия/выносливость нормированы). Дополнительно —
`VecNormalize` (obs + reward) для устойчивости PPO.

## Действия и награда

- **Действие:** `MultiDiscrete([3, 2])` — руление `{−Δ°, прямо, +Δ°}` × рывок `{нет, да}`.
  SB3 PPO поддерживает `MultiDiscrete` (MultiCategorical). Рывок при пустой
  `stamina` игнорируется (движение на `v_snake`).
- **Reward:**
  - `+10` съедена курица.
  - **Потенциальный шейпинг (PBRS):** `F = γ·Φ(s′) − Φ(s)`, где `Φ = −dist_to_nearest_chicken / ray_range` (ограничена). PBRS **доказуемо не меняет оптимальную политику** и телескопируется → любой замкнутый цикл даёт ~0, поэтому «зависнуть рядом с курицей и фармить запах» невыгодно (наивный `+за близость` — фармится, поэтому запрещён). На терминале `Φ=0`; при смене «ближайшей курицы» (съел/спавн/убежала) скачок `Φ` не выплачиваем (обнуляем `F` в этот шаг).
  - `−10` смерть, эпизод обрывается.
  - Пошаговый штраф с голодом: `−0.01 · (1 + (1 − E/energy_max))` — чем голоднее, тем дороже топтаться (делает наблюдаемую энергию действенной; смерти от голода нет).
  - анти-верчение: PPO-энтропия + опц. крошечный штраф за долгий поворот в одну сторону.
  - рывок сам по себе не штрафуется — его ограничивает выносливость.
  - **Баланс:** при `γ<1` растянуть `−0.01` дороже, чем умереть `−10` сейчас → нет
    стимула к суициду. `T`, ожидаемый темп поедания и коэффициенты подбираем так,
    чтобы шаговый штраф не забивал сигнал `+10` (проверяем по логам среднего reward).

## Обучение и наблюдение

- **Train:** 8–16 параллельных окружений через **`SubprocVecEnv`** (истинный параллелизм
  по CPU-ядрам), каждый — новый рандомный мир. `device="cpu"` (см. ниже). Headless.
  Callback раз в N шагов пишет чекпоинт + статистику (эпизоды, средний reward, съедено
  кур, смертей, средняя длина).
- **Устройство:** обучаем на **CPU**. SB3 `get_device` официально поддерживает только
  `cpu`/`cuda` — `device="auto"` не выберет MPS; и для крошечного MLP CPU+`SubprocVecEnv`
  быстрее MPS (оверхед запуска ядер на маленьких батчах). MPS — только опциональный
  бенчмарк, не основной план. *(Fallback самописного PPO — на случай реально сломанного
  пина версий SB3/gym/torch, а не ради MPS.)*
- **Политика:** `MlpPolicy` + `VecFrameStack(4)` (память под POMDP). Если верчение/забывание
  тела останутся проблемой — апгрейд на `RecurrentPPO` (sb3-contrib, поддерживает
  `MultiDiscrete`). Отмечено как запасной путь, не стартовый.
- **Curriculum (опц.):** полностью случайный размер мира повышает дисперсию возврата
  (шумнее сходимость). При необходимости — лёгкий curriculum (сначала мельче/меньше
  препятствий). Стартуем без него, следим за дисперсией по логам.
- **Watch:** грузит последний чекпоинт, гоняет одну змею в свежем рандомном мире с полной
  отрисовкой и оверлеем сенсоров (лучи, «облако запаха»). Политика — `deterministic=True`.
  Клавиши: пауза, новый мир, показать/скрыть сенсоры.
- **«Останови и посмотри»:** train постоянно сохраняет чекпоинт → в любой момент запускаешь watch на актуальной модели.
- **Save/load/reset:** модель — файл чекпоинта SB3 (+ статистика `VecNormalize`); `--reset` с нуля, `--model path` грузит конкретный.

## Архитектура (модули)

Изолированные части с чёткими границами, каждую можно тестировать отдельно.

| Модуль | Ответственность | Зависит от |
|---|---|---|
| `config.py` | Все константы + проверка инвариантов достижимости | — |
| `world.py` | Состояние и физика: змея, куры, препятствия, тор (ближайший образ), swept-collision, `step()` | numpy, config |
| `worldgen.py` | Рандомная генерация (размер, камни/деревья, стартовые куры) с rejection sampling | numpy, config |
| `sensors.py` | Рей-кастинг + запах → вектор 42. Читает **минимальный интерфейс мира** (геометрия препятствий, куриц, тела, размер) — «чистая по входам», не самодостаточная | numpy, config |
| `env.py` | Gymnasium: obs ← sensors, action → world.step, reward (PBRS), `terminated`/`truncated`, reset; assert инвариантов | world, sensors, config |
| `train.py` | PPO (SB3), `SubprocVecEnv` + `VecFrameStack` + `VecNormalize`, чекпоинты, логи | env, SB3 |
| `render.py` | pygame: мир + оверлей сенсоров (лучи, запах) + статистика | pygame, config |
| `watch.py` | Загрузка модели, прогон в новом мире с рендером (`deterministic`), клавиши | env, render |
| `__main__.py` | CLI: `train` / `watch` / `--reset` / `--model` | всё |

**Почему SB3, а не самописный PPO:** PPO легко написать неправильно (GAE, клиппинг,
нормализация) и мучительно отлаживать. SB3 — отлаженная реализация, меньше кода,
из коробки save/load, векторные env, `VecFrameStack`/`VecNormalize`. Env и рендер
кастомные. **Fallback:** сломанный пин версий → компактный самописный PPO (~250 строк)
в том же `train.py`; интерфейс `env`/чекпоинтов не меняется.

## Тестирование

По одному запускаемому `assert`-тесту на нетривиальную логику, без фреймворков:
- тор: ближайший образ (`d = ((a−b+L/2) mod L) − L/2`) и **луч через шов** видит объект на дальней стороне;
- рей-кастинг: луч видит объект на известной позиции; пустой луч кодируется (dist=1, тип=нули);
- блокировка запаха препятствием (LoS = 0 за камнем);
- **туннелирование:** быстрая голова (рывок) против тонкого препятствия/тела → смерть засчитана (swept);
- детект столкновения с собой (с порогом-пропуском по дуге);
- «съедание» курицы (в `eat_radius`);
- инвариант достижимости (1)+(2)+(3) из конфига;
- прицельная точность (I4): при выполнении (3) есть действие, наводящее в «коридор поедания»;
- расход/восстановление выносливости (рывок при пустой `stamina` не ускоряет);
- **PBRS не фармится:** замкнутый цикл состояний даёт суммарный `F ≈ 0` (телескопирование);
- Gymnasium `check_env` проходит; `reset(seed=...)` детерминирован.

## Явно вне скоупа (YAGNI)

- Плавно-переменная скорость змеи (только два уровня: обычная / рывок).
- Реалистичная диффузионная сетка запаха (взяли упрощённую `1/(1+r)` + LoS).
- Смерть от голода (по ТЗ отключена; голод действует через шаговый штраф).
- Веб-визуализация (pygame достаточно).
- Мультиагентность / несколько змей.
- `RecurrentPPO`, curriculum — как запасные апгрейды, не в стартовом объёме.
