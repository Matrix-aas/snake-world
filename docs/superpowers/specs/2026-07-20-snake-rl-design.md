# Snake-RL — обучаемая змея в непрерывном 2D-мире

**Дата:** 2026-07-20
**Статус:** дизайн утверждён, готов к плану реализации

## Идея

2D-мир как полноценная игра: змея с непрерывным углом движения (не только
4 направления), мир замыкается тором (лево↔право, верх↔низ). Змеёй управляет
ИИ, обученный с подкреплением (RL). Два режима: **train** (наблюдаем, как M1
оптимально обучает змею) и **watch** (смотрим на обученную змею в свежем
рандомном мире). Миры всегда случайные — змея учится выживать в любом.

Змея не знает карту — только органы чувств: зрение (рей-кастинг, 7 лучей) и
обоняние (запах от куриц, слабеет с расстоянием, блокируется препятствиями).
Ест куриц (растёт), умирает от столкновения с камнем/деревом или с собой.

## Решения (согласованы с пользователем)

- **Стек:** Python + PyTorch (MPS на M1) + PPO (Stable-Baselines3) + Gymnasium + pygame.
- **Управление:** дискретное руление — 3 действия (поворот −Δ° / прямо / +Δ°). Любой угол достижим накоплением.
- **Голод:** только мотивация в reward, **не убивает** (строго по ТЗ). Смерть — только от столкновений.
- **Обоняние:** упрощённое — сила ~ 1/расстояние, блокируется препятствиями (проверка прямой видимости).
- **Наблюдение за обучением:** train крутится headless на полной скорости с текстовой статистикой; живая картинка — отдельный watch-режим на актуальном чекпоинте.

## Архитектура (модули)

Изолированные части с чёткими границами, каждую можно тестировать отдельно.

| Модуль | Ответственность | Зависит от |
|---|---|---|
| `world.py` | Состояние и физика: змея, куры, препятствия, тор-мир, `step()` | numpy |
| `worldgen.py` | Рандомная генерация мира (размер, камни/деревья, стартовые куры) | numpy |
| `sensors.py` | Рей-кастинг (зрение) + запах → вектор наблюдения. Чистая функция | numpy |
| `env.py` | Обёртка Gymnasium: obs ← sensors, action → world.step, reward, done, reset | world, sensors |
| `train.py` | PPO (SB3), N параллельных рандомных миров, чекпоинты, логи | env, SB3 |
| `render.py` | Отрисовка pygame: мир + оверлей сенсоров (лучи, запах) + статистика | pygame |
| `watch.py` | Загрузка модели, прогон в новом мире с рендером, клавиши | env, render |
| `__main__.py` | CLI: `train` / `watch` / `--reset` / `--model` | всё |

**Почему SB3, а не самописный PPO:** PPO легко написать неправильно (GAE,
клиппинг, нормализация) и мучительно отлаживать. SB3 — отлаженная реализация,
меньше кода, из коробки save/load и векторные окружения. Env и рендер всё равно
кастомные. **Fallback:** если всплывут проблемы совместимости версий SB3 с
torch/gymnasium/numpy — переходим на компактный самописный PPO (~250 строк) в
том же `train.py`; интерфейс `env` и чекпоинтов не меняется.

## Мир и физика

- **Континуальный тор:** координаты float, `x % W`, `y % H`. Размер рандомный в диапазоне (напр. 60–100 у.е.).
- **Змея:** голова движется с постоянной скоростью под углом `heading`. Тело — след из точек: пишем траекторию головы, ставим сегменты через фиксированный интервал. Рост от еды = увеличение целевой длины. Столкновение с собой = голова пересекает сегмент тела (пропускаем несколько ближайших к голове — они всегда рядом).
- **Препятствия (камни/деревья):** статичные круги разного радиуса. Поведение идентично (оба убивают при касании головой) — одна сущность с флагом типа только для отрисовки. *ponytail: не плодить два класса ради разных спрайтов.*
- **Куры:** медленное блуждание (случайное направление, редкая смена) + лёгкое убегание, если змея близко. Максимум 5, новые появляются со временем в свободных точках. Съедена = голова в радиусе курицы → змея растёт, энергия восполняется.
- **Смерть:** только касание головой препятствия или собственного тела. О «стену» не умирает (мир-тор).

## Органы чувств (вектор наблюдения)

Всё эгоцентрично (в системе координат головы). Змея карту не знает.

- **Зрение — 7 лучей** веером вперёд (FOV ±90°). Каждый луч: нормированная дистанция до ближайшего попадания + тип one-hot `{препятствие, курица, своё тело}`. → 7 × 4 = 28 чисел.
- **Обоняние:** запах каждой курицы ~ `1/расстояние`, блокируется препятствием (проверка прямой видимости — «за камнем не пахнет»), суммируется. Даём: интенсивность в точке головы + локальный градиент (2 компоненты, повёрнутые в систему головы). → 3 числа.
- **Проприоцепция:** уровень энергии (голод), длина тела. → 2 числа.

Итого ~33 float. Есть слепые зоны (сзади только «тело»-луч), запах даёт направление, но не точную позицию.

## Действия и награда

- **Действие:** дискретное, 3 варианта — `−Δ°` / `прямо` / `+Δ°`.
- **Reward:**
  - `+10` съедена курица
  - `+` маленький шейпинг за рост запаха / сокращение дистанции до ближайшей курицы (искать, а не блуждать)
  - `−10` смерть, эпизод обрывается
  - `−0.01` за шаг (голод/время — стимул не тупить; смерти от голода нет)
  - анти-верчение: PPO-энтропия + опц. крошечный штраф за долгий поворот в одну сторону

## Обучение и наблюдение

- **Train:** 8–16 параллельных векторных окружений, каждый — новый рандомный мир. Headless (быстро, грузит M1). Callback раз в N шагов пишет чекпоинт + статистику (эпизоды, средний reward, съедено кур, смертей).
- **Watch:** грузит последний чекпоинт, гоняет одну змею в свежем рандомном мире с полной отрисовкой и оверлеем сенсоров (лучи, «облако запаха»). Клавиши: пауза, новый мир, показать/скрыть сенсоры.
- **«Останови и посмотри»:** train постоянно сохраняет чекпоинт → в любой момент запускаешь watch на актуальной модели.
- **Save/load/reset:** модель — файл чекпоинта SB3; `--reset` с нуля, `--model path` грузит конкретный.

## Окружение и версии

- Python 3.13 (venv) — стабильная связка с torch/SB3. Версии пакетов пиним в `requirements.txt`; фактические версии проверяем при установке.
- Apple Silicon: PyTorch MPS для сети; векторные env крутятся на CPU (сеть маленькая, узкое место — не матмуль).

## Тестирование

По одному запускаемому `assert`-тесту на нетривиальную логику, без фреймворков:
- тор-обёртка координат,
- рей-кастинг (луч видит объект на известной позиции),
- блокировка запаха препятствием,
- детект столкновения с собой,
- «съедание» курицы.

## Явно вне скоупа (YAGNI)

- Переменная скорость змеи (действие только руление; скорость константа).
- Реалистичная диффузионная сетка запаха (взяли упрощённую 1/r).
- Смерть от голода (по ТЗ отключена).
- Веб-визуализация (pygame достаточно).
- Мультиагентность / несколько змей.
